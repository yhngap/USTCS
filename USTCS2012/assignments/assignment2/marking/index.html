Marking scheme and appeal procedure

Grade Checking

  • We have uploaded all test cases to ZINC and regraded your latest submissions.
  • If you want to appeal your grade: you MUST read this whole page and follow all procedures correctly. Your appeal will be dropped otherwise and will NOT be processed.
  • Please ALWAYS check your scores at ZINC. Scores in ZINC exclude penalties resulted from the request for fixing/late submission, for instance.
  • Score after penalty can be checked here. It shows you the total score minus any penalty that needs to be applied.
  • For individual test case result and score, please find the updated pa2 grading result on ZINC.
  • We will perform plagiarism checking at a later time and marks will be adjusted for those who are caught.
  • Statistics: mean is 72.7 (out of 90) and SD is 23.8 (for submitted works only)

Marking scheme

  • The first 3 test cases are given before the deadline that are not graded, 45 hidden test cases are used for grading post-deadline.
  • Each test case is worth 2 marks. No partial credit.
  • Total for all 45 cases is 90 marks.
  • Memory leak checking is applied on ALL test cases using the -fsanitize=address,leak,undefined flags compilation flag (see submission guide).

Trying the Test Cases

You can see all the test case results on ZINC in the latest grading report. If you want to try the test case on your own computer, you may do the following.

  • Download and extract the files in this test package
  • Add them to VSCode or whatever IDE you use. Note that this is a Makefile project.
  • Download your latest submission from ZINC
  • Extract your submitted files and add them to the project.
  • Make and run the test cases.
  • Check the output and read the code of each test case to understand what is being tested. You may find the expected from the ZINC grading report.

Notes and appeal procedure

It differs from PA1. Please read carefully.

  • Runtime errors, crashes, and inconsistent output due to incorrect memory access / memory allocation / memory deallocation / missing variable initialization / missing return values / out-of-bound array access / etc. may not occur all the time on all machines. However, even you may not encounter the error/crash when you test your program, if your program crashes during any of our testing on ZINC, the corresponding test cases will be considered as failed and will receive 0 marks as your code is indeed faulty.
  • If you somehow get a correct output on your own computer but you get a different output on ZINC, it is likely your code is faulty. Refer to the previous comments for possible causes. Verify your code to make sure they aren't faulty and won't produce any runtime error before you appeal those cases. Compare your solution with the sample solution yourself if needed.
  • We can run the test cases again for you (after you have verified them yourself), but all grading shall be done objectively in an automatic and consistent way on the very same machine (the ZINC grading machine, not any other machine including yours or ours). However, if your program crashes or gives unreliable results, it is guaranteed that the problem is somewhere in your code, and marks will not be given. For grading, only the result of your program on the ZINC machine matters.
  • It is possible that you get a lower mark as we re-run the test cases for you and found additional crashes/mistakes. We re-run all test cases (not just the ones you want to re-run) for all appeals. So proceed carefully at your own risk, not just blindly appeal your grade anyway.
  • It is mentioned in the assignment instruction that "It is required that your submissions can be compiled and run successfully in our online autograder ZINC. If we cannot even compile your work, it won't be graded. Therefore, for parts that you cannot finish, just put in dummy implementation so that your whole program can be compiled for ZINC to grade the other parts that you have done.."
  • For compilation errors/memory leak/incorrect program print outs, you get 0 mark for that test case. To appeal, you should find out how to fix the errors yourself, and then clearly tell us how in the email. If the fix is simple, we will then issue an approval email, detailing the scores you can get from your proposed fix adjusted by the penalty. After receiving the email, you can resubmit your fixed work to ZINC by the deadline designated in the email. We will regrade it with a 30% penalty on the total scores that you gained from that SINGLE fixing case in the appeal. (With a minimum penalty of 10 marks per fixing case/error). The fix should fix simple errors which have large impact on your scores only and must not change the logic of your program. Please do the math before appealling.
  • Please understand that, for fairness, we usually do NOT allow any student to change their submitted code, except only the special case mentioned above.
  • If you are certain that there is a grading mistake after reading this whole page, reading the assignment description/FAQ/Piazza, and reading and trying the test cases yourself, email to mhwongah@connect.ust.hk with the following information. Appeals with incomplete information will not be processed.
    • Email subject must be COMP2012 Fall 2020 PA2 Appeal
    • Email body contains all of the following:
      • Your full name as shown on your student ID card
      • Your HKUST itsc username
      • Your student ID number
      • Test cases for which you want to appeal
      • Justification/fixes (if any) for your appeal case
  • Do not attach any .exe/executable file in the email as they will be blocked by the email server.
  • Appeal deadline is Dec 2nd (Wednesday) 23:59pm. No late appeals will be processed.
  • Due to the large number of students that we have, please allow up to 72 hours for your appeal email to be processed and replied. For those requesting for fixing, please resubmit the code by designated deadline otherwise your request will not be entertained and no penalty will be applied. If you do not receive a reply in 72 hours, please re-send your email.
  • Finalized scores will only be uploaded to Canvas after all appeals are processed.
