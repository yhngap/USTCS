Lab 7 More on Inheritance

Vehicle System

Objective

This lab is about using different types of inheritance. You will complete the implementation of a base class and several derived classes. The following skills will be employed and illustrated:

  • Public, protected and private inheritance
  • Directed and indirected inheritance

Description

The figure below depicts the structure of four classes: Engine, Vehicle, DeliveryCar and PoliceCar:

The figure above is a UML diagram. The diagram is pretty straight-forward, but please be aware that: the plus "+" denotes public members, while the minus "-" denotes private members. If you want, you may learn about the basics of the diagram here.

Class Description

There are 4 classes, Engine, Vehicle, DeliveryCar, and PoliceCar.

In this part, Engine is the abstract base class.

Vehicle derives from class Engine using private inheritance.

DeliveryCar derives from class Vehicle using protected inheritance. Besides data members and member functions inherited from Vehicle, DeliveryCar also has its own private data member: int chargePerMile, which represents the delivery charge per mile. Two member functions: setCharge() and getCharge() are used to access and mutate the data member chargePerMile.

PoliceCar is a derived class of Vehicle, public inheritance is used. Besides data members and member functions inherited from Vehicle, PoliceCar also has its own private data member: bool inAction, which indicates whether the police car is in action. Two member functions: getInAction() and setInAction() are used to access and mutate the data member inAction.

You are given the class definition of the Engine class (Engine.h), the class definition of Vehicle (Vehicle.h) and the class definition of PoliceCar (PoliceCar.h). But you need to make some modification.

A main function (main.cpp) is also provided for testing.

More Info

Casting Base Class Pointers to Derived Class Pointers:

  • Object of a derived class
    • Can be treated as an object of the base class.
    • Reverse not true - base class objects not a derived-class object.

Example:

  • DeliveryCar and PoliceCar class derived from the Vehicle class
  • We can use pointer of type Vehicle to reference a DeliveryCar and PoliceCar object
    • Vehicle *c = new Vehicle(12,Silver,12000);
    • PoliceCar *pc = new PoliceCar(8, Black, 100000, false);
    • c = pc; //Treat a PoliceCar as a Vehicle
Vehicle *c = new Vehicle(12,Silver,12000);
