Lab 4 Inheritance Basics

Introduction

Let's try something different in this lab.

You will be given a class diagram, description of the classes, and a main.cpp that uses those classes. You will create the corresponding class header and source files yourself, to practice creating classes with basic inheritance from scratch.

Description

There are 4 classes in total:

The top-level base class is Animal. Dog inherits from Animal while Corgi and GreatDane inherit from Dog.

Animal

You should declare the class in animal.h and define the member functions in animal.cpp. You can see that animal.h is included in main.cpp.

It should have only one private data member name of the string type. It has no other data member.

It has only one constructor Animal(string name) as you can see that is invoked in the main function. It simply sets the name of the animal upon construction of an animal object.

It can talk and eat. (they are also invoked in the main function) We also are able to retrieve its name via the getName function. You need to determine their function headers yourself. Check the main.cpp to see how they are used, and derive what they need to output.

Dog

You should declare the class in dog.h and define the member functions in dog.cpp. You can see that dog.h is included in main.cpp.

It should NOT have any data member defined in Dog itself. Of course, it should inherit name from the parent class.

It has only one constructor Dog(string name) as you can see that is invoked in the main function. It simply sets the name of the dog upon construction of a dog object. You need to use the member initialization list (MIL) here.

It can talk, eat, and bark. (they are also invoked in the main function) You need to determine their function headers yourself. Check the main.cpp to see how they are used, and derive what they need to output. talk and eat are already defined in the parent class, therefore, you need to define them again (i.e. override) in the Dog class to give them new implementations.

Note: getName should not be defined in the Dog class itself - it should be inherited from the parent class.

Corgi

Note: talk, eat, and getName should not be defined in the Corgi class itself - it should be inherited from the parent class.
