COMP 2012 Object-Oriented Programming and Data Structures

Lab 5 More on Inhertiance


Introduction

Let's play with cats this time. Meow~

You will be given a class diagram, description of the classes, a main.cpp that uses those classes, and some of the class header files. You will create the missing class header and source files yourself, to practice creating classes with inheritance from scratch.

Description

There are 4 classes in total:

The top-level base class is Cat. FatCat inherits from Cat while UltraFatCat inherits from FatCat. Toy is a separate class. A Cat owns a Toy.

You should already have some idea on how to create and design simple classes after lab 4, so we are not going to repeat the obvious here.

Cat

The cat.h is already given in the starting code. You should not modify or submit that. Please follow the instructions in the header file and check the sample output to implement the Cat class in cat.cpp.

FatCat

You should declare the class in fatcat.h and define the member functions in fatcat.cpp. You can see that fatcat.h is included in main.cpp.

It should NOT have any data member defined in FatCat itself.

Compared to its parent class, it has a new function eatBigFish which increases the fullness by 20. It also redefines play which decreases fullness by 8 only.

UltraFatCat

You should declare the class in ultrafatcat.h and define the member functions in ultrafatcat.cpp. You can see that ultrafatcat.h is included in main.cpp.

It should NOT have any data member defined in UltraFatCat itself.

Compared to its parent class, it has a new function eatFishBuffet which increases the fullness by 30. It also redefines play which decreases fullness by 5 only.

Toy

A cat owns a toy. The toy.h is already given in the starting code. You should not modify or submit that. Please follow the instructions in the header file and check the sample output to implement the Toy class in toy.cpp.
