Oct. 31

Q: In the project description states: Only a single supervisor grade should be given for each graded requirement even if an FYP has more than one supervisor. Is this a constraint that can be added when we create the table?
A: No. It needs to be implemented in the application.

Q: If a project group is not assigned to an FYP, then are the "assignedFYP", "groupCode" and "reader" attributes in the ProjectGroup relation all null?
A: Yes.

Q: Are the default values for "proposalReport", "progressReport", "finalReport" and "presentation" 0 or null in the RequirementGrades relation?
A: They are null until given a value.

Q: Part 1: Do we need to drop all tables before creating them like what is done in the lab DB files?
A: No.

Q: Query 1: We need to display two supervisors' names and the title and category of the FYP. Is it okay to display the two supervisors' names in two rows.
A: Yes.

Q: Query 2: Is there anything I should do for a project group that has been assigned to an FYP, but has no reader assigned yet? Can I just show the null value for the reader's name?
A: You can just show null.

Q: Query 3: It asks to find the number of project groups for each FYP category but asks to order by the number of FYPs. Does it mean that the result returned should include the FYP category, the number of project groups and the number of FYPs?
A: No, its a typo. It should be "order by number of project groups", not "order by number of FYPs".

Q: Query 4: “Find the title, supervisor name, names of the students in the project group...”. Here, one FYP title can only join with one ProjectGroup, but inside the ProjectGroup, we can have multiple students. Does this mean the query returns tuples with respect to each student that belongs to the same project group?
A: Yes.

Oct. 26

Q: Do we need to also set the null value constraints for all attributes? If so, then how can we derive them? It doesn't seem very obvious when referring to both the Task 2 and Task 1 description. For example, can either the description or otherRequirements of FYP be null?
A: We believe it is possible to determine whether an attribute value can be null from the description and by applying a little bit of common sense. For the description of an FYP, how can students decide whether to select an FYP if there is no description of it? For other requirements, do you think that every FYP needs to have some other requirements specified for it? For faculty room number, do you know of any faculty that does not have an office room number? Also, the first sentence of the paragraph that describes these attributes states "Required FYP information ..." Usually "required" means must have a value.

Q: Are we supposed to add constraints that can be derived from our common sense, but not specified in the project description? For example, do I have to add a constraint on CSEStudent's name attribute as normal people won't have a digit in their names?
A: Yes, but please use common sense. If you can find a regular expression on the web that works for all people's names, then by all means use it. But we bet that you cannot find such a regular expression because people's names can vary and who says it cannot contain a number or a hyphen or a period, etc.? So, the best we think you can do for a name is simply to check that it is not null.

Q: The project description specifies the categories that an FYP can have. Can there be any other category of FYP other than those in the list (Artificial Intelligence, Computer Graphics, ..., Theory, Vision and Graphics)?
A: These are the only FYP categories allowed.

Q: Can an FYP exist without having a category? For example, what would happen to FYPs that have ‘Artificial Intelligence’ as their categories if the FYP category ‘Artificial Intelligence’ is deleted? Would they still exist in the database and wait for another closest category to be assigned?
A: An FYP cannot exist with no value for the category. This is part of the required information for an FYP. If a category is deleted, then the FYP must be deleted. But why would you ever want to delete an FYP category in the first place?

Q: For query 4, if a project group has been assigned to an FYP that they have not indicated an interest in or have specified an interest with priority equal to 1, then will they be excluded from the query result?
A: Yes.
